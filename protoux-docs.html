ProtoUX

An experimental, HTML-based wireframe and prototype framework for designers.

Getting Started

Designers

  1. Use the latest version of Chrome.
  2. Enable the experimental chrome flag:
    chrome://flags/#enable-experimental-web-platform-features
  3. Download the ProtoUX.zip and extract into a directory of your choosing. Copy this directory path.
  4. Setup IIS for a localhost server.
  5. Create your wireframes and prototype! Sublime and Notepad++ are great, free editors.

Participants

  1. Use the latest version of Chrome.
  2. Enable the experimental chrome flag:
    chrome://flags/#enable-experimental-web-platform-features
  3. Bug your designer for the URL to the prototype.


Importing Partial .html Files

The cleanest method for importing partial HTML files into your page:

<link is="ux-link" id="[*]" href="[*]" data-layout="[*]" rel="import">

Be sure to replace the [*] with a unique id for each import, the proper directory path to the import, and any optional layout class you want to be applied to the import.

Building Partial .html Files

The root node of your partial .html files should have this structure:

<div class="wrap">
<-- Your HTML -->
</div>


File Structure

The root directory of the ProtoUX zip file is shown below. Don't remove or change the required files or folders. Otherwise, go nuts however you want to organize your prototype.

Layout

With standard HTML:

<div class="row">
<div class="col span_#_of_#">
<-- Your HTML -->
</div>
</div>

With link attributes:

<div class="row">
<link is="ux-link" id="[*]" href="[*]" data-layout="[col span_#_of_#]" rel="import">
</div>


Annotations

An example annotation.

Include any annotations within any parent container:

<ux-annotation>Your annotation.</ux-annotation>

ProtoUX Control Bar

The ProtoUX control bar is included via the custom "ux-controlbar" tag at the bottom of the index.html file.

Toggle the bar visibility with the tilde (~) key.

IIS Setup

(not as scary as it looks)

  1. Windows search for: "Internet Information Services (IIS) Manager" and open.
    1. If you do not have IIS installed, it’s a windows feature that you can turn on:
    2. Windows search for “Turn Windows features on or off”
    3. Ensure that “internet information services” is checked
  2. Click the expand triangle underneath “connections” on the left until you see “sites”.
  3. Click "sites"
  4. Click "Add Web Site..." on the right "Actions" menu
  5. Choose a site name
  6. Paste the directory path where you prototype files live into the “physical path” input.
  7. Click "Connect as...", select "Specific user:", click "Set...", enter your full TEN\Username and password.
  8. Click "Test Connection" and ensure all items in the results list is green/checked
  9. Enter a port number that isn’t in use (84 used arbitrarily for this demo)
  10. Click “OK”
  11. Click on "Browse *:84 (http)" on the right side "actions" menu to ensure the site is started and working.
